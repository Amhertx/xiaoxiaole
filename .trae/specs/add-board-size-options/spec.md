# 棋盘尺寸选项功能规格文档

## Why
当前游戏仅支持固定的8x8棋盘尺寸。为了适应不同设备和用户偏好，需要增加6x6棋盘选项。PC端用户可以选择6x6或8x8棋盘，移动端由于屏幕空间限制，仅提供6x6棋盘选项，以获得更好的游戏体验。

## What Changes
- 新增棋盘尺寸配置系统，支持6x6和8x8两种尺寸
- PC端提供棋盘尺寸选择界面，允许用户在游戏开始前选择棋盘大小
- 移动端自动锁定为6x6棋盘，不显示选择选项
- 修改游戏逻辑，使棋盘尺寸可配置而非硬编码
- 调整UI布局，确保两种尺寸下都有良好的视觉效果

## Impact
- Affected specs: 游戏棋盘系统、用户界面
- Affected code: 
  - `src/types/game.ts` - 移除硬编码的BOARD_SIZE，改为可配置
  - `src/stores/gameStore.ts` - 支持动态棋盘尺寸
  - `src/utils/gameLogic.ts` - 所有函数支持动态尺寸参数
  - `src/components/GameBoard.vue` - 根据尺寸动态渲染棋盘
  - `src/components/GameHeader.vue` - 添加棋盘尺寸选择UI
  - `src/App.vue` - 可能需要响应式布局调整

## ADDED Requirements

### Requirement: 棋盘尺寸配置系统
系统应支持可配置的棋盘尺寸，包括6x6和8x8两种选项。

#### Scenario: 棋盘尺寸定义
- **WHEN** 系统初始化时
- **THEN** 系统应定义支持的棋盘尺寸类型（6x6和8x8）
- **AND** 每种尺寸应有对应的配置参数

#### Scenario: 棋盘尺寸存储
- **WHEN** 用户选择棋盘尺寸后
- **THEN** 系统应将选择的尺寸保存到游戏状态中
- **AND** 所有游戏逻辑应使用当前配置的尺寸

### Requirement: PC端棋盘尺寸选择
PC端用户应能够在游戏开始前选择棋盘尺寸。

#### Scenario: 显示尺寸选择界面
- **WHEN** PC端用户进入游戏时
- **THEN** 系统应显示棋盘尺寸选择界面
- **AND** 提供6x6和8x8两个选项
- **AND** 默认选中8x8尺寸

#### Scenario: 选择并开始游戏
- **WHEN** PC端用户选择尺寸并点击开始
- **THEN** 系统应使用选择的尺寸初始化棋盘
- **AND** 开始游戏

#### Scenario: 记住用户选择
- **WHEN** 用户选择棋盘尺寸后
- **THEN** 系统应将选择保存到localStorage
- **AND** 下次打开游戏时自动选中上次的选择

### Requirement: 移动端棋盘尺寸限制
移动端应仅使用6x6棋盘，不提供选择选项。

#### Scenario: 移动端检测
- **WHEN** 用户在移动设备上访问游戏时
- **THEN** 系统应检测设备类型
- **AND** 自动设置为6x6棋盘

#### Scenario: 隐藏尺寸选择
- **WHEN** 在移动端显示游戏界面时
- **THEN** 系统应隐藏棋盘尺寸选择UI
- **AND** 直接显示6x6棋盘

### Requirement: 游戏逻辑适配
所有游戏逻辑应支持动态棋盘尺寸。

#### Scenario: 棋盘初始化
- **WHEN** 使用指定尺寸初始化棋盘时
- **THEN** 系统应创建对应大小的棋盘
- **AND** 确保初始棋盘没有匹配

#### Scenario: 匹配检测
- **WHEN** 检测匹配时
- **THEN** 系统应根据当前棋盘尺寸进行检测
- **AND** 边界检查应使用正确的尺寸值

#### Scenario: 特殊道具效果
- **WHEN** 触发特殊道具效果时
- **THEN** 系统应根据当前棋盘尺寸计算影响范围
- **AND** 确保不超出棋盘边界

### Requirement: UI响应式适配
游戏界面应根据棋盘尺寸进行响应式调整。

#### Scenario: 棋盘布局
- **WHEN** 渲染棋盘时
- **THEN** 系统应根据棋盘尺寸调整单元格大小和间距
- **AND** 确保棋盘在屏幕中居中显示

#### Scenario: 元素尺寸
- **WHEN** 渲染游戏元素时
- **THEN** 系统应根据棋盘尺寸调整元素大小
- **AND** 6x6棋盘的元素应比8x8的元素更大

#### Scenario: 分数显示适配
- **WHEN** 显示分数时
- **THEN** 系统应根据棋盘尺寸分别记录和显示最高分
- **AND** 6x6和8x8棋盘应有独立的最高分记录

## MODIFIED Requirements

### Requirement: 游戏棋盘系统（修改自原spec）
系统应提供一个可配置尺寸的游戏棋盘（6x6或8x8），支持元素交换、匹配检测和自动填充。

#### Scenario: 初始化棋盘
- **WHEN** 游戏开始时
- **THEN** 系统应根据配置的尺寸生成棋盘，随机填充 5-6 种不同颜色和形状的几何图形
- **AND** 确保初始棋盘没有已匹配的元素

## REMOVED Requirements
无
