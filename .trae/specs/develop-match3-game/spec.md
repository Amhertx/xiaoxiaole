# 消消乐小游戏开发规格文档

## Why
开发一个基于 Vue3 + Vite + Pinia 的经典三消游戏，提供休闲娱乐体验，帮助用户放松心情。选择无限模式降低开发复杂度，使用简单几何图形减少素材依赖，同时加入特殊道具和连击特效提升游戏趣味性。

## What Changes
- 创建基于 Vue3 的消消乐游戏核心系统
- 实现经典三消游戏机制（交换、匹配、消除、下落）
- 设计简单几何图形的游戏元素系统
- 开发无限模式游戏流程
- 实现特殊道具系统（消除4个及以上产生特殊道具）
- 添加连击系统和视觉特效
- 使用 Pinia 进行状态管理
- 响应式设计，支持不同屏幕尺寸

## Impact
- 新增游戏核心模块
- 新增游戏状态管理
- 新增游戏组件库
- 新增游戏工具函数
- 新增样式系统

## ADDED Requirements

### Requirement: 游戏棋盘系统
系统应提供一个 8x8 的游戏棋盘，支持元素交换、匹配检测和自动填充。

#### Scenario: 初始化棋盘
- **WHEN** 游戏开始时
- **THEN** 系统应生成一个 8x8 的棋盘，随机填充 5-6 种不同颜色和形状的几何图形
- **AND** 确保初始棋盘没有已匹配的元素

#### Scenario: 元素交换
- **WHEN** 用户点击并拖动一个元素到相邻位置
- **THEN** 系统应交换这两个元素的位置
- **AND** 检查是否形成匹配
- **AND** 如果没有匹配，则将元素交换回原位置

#### Scenario: 匹配检测
- **WHEN** 元素交换后
- **THEN** 系统应检测是否存在 3 个或以上相同元素的连线
- **AND** 标记所有匹配的元素

#### Scenario: 元素消除和下落
- **WHEN** 检测到匹配后
- **THEN** 系统应消除所有匹配的元素
- **AND** 上方元素自动下落填充空位
- **AND** 从顶部生成新元素填充剩余空位
- **AND** 递归检测新的匹配

### Requirement: 游戏元素系统
系统应提供基于简单几何图形的游戏元素，每种元素具有独特的颜色和形状。

#### Scenario: 元素类型
- **WHEN** 游戏初始化时
- **THEN** 系统应创建 5-6 种不同的元素类型
- **AND** 每种类型具有不同的颜色（红、蓝、绿、黄、紫、橙等）
- **AND** 每种类型具有不同的几何形状（圆形、方形、三角形、菱形、五边形、六边形等）

#### Scenario: 元素渲染
- **WHEN** 渲染游戏元素时
- **THEN** 系统应使用 CSS 或 Canvas 绘制几何图形
- **AND** 元素应具有平滑的动画效果

### Requirement: 无限模式游戏流程
系统应提供无限模式，玩家持续游戏直到无法移动或主动退出。

#### Scenario: 游戏开始
- **WHEN** 用户点击开始游戏
- **THEN** 系统应初始化棋盘
- **AND** 重置分数和连击计数器
- **AND** 开始游戏循环

#### Scenario: 分数计算
- **WHEN** 消除元素时
- **THEN** 系统应根据消除的元素数量计算分数
- **AND** 基础分数 = 消除数量 × 10
- **AND** 连击加成 = 连击次数 × 5
- **AND** 总分 = 基础分数 + 连击加成

#### Scenario: 游戏结束
- **WHEN** 棋盘上没有可行的移动时
- **THEN** 系统应检测游戏结束
- **AND** 显示最终分数
- **AND** 提供重新开始选项

### Requirement: 特殊道具系统
系统应在消除 4 个或以上元素时生成特殊道具，提供额外的消除能力。

#### Scenario: 生成特殊道具
- **WHEN** 消除 4 个相同元素时
- **THEN** 生成一个"炸弹"道具，可以消除周围 3x3 范围的元素
- **WHEN** 消除 5 个相同元素时
- **THEN** 生成一个"超级炸弹"道具，可以消除整行或整列的元素

#### Scenario: 使用特殊道具
- **WHEN** 特殊道具被匹配或交换时
- **THEN** 触发特殊消除效果
- **AND** 显示特殊特效动画
- **AND** 计算额外分数

### Requirement: 连击和特效系统
系统应提供连击机制和视觉特效，增强游戏体验。

#### Scenario: 连击检测
- **WHEN** 连续消除发生时
- **THEN** 系统应记录连击次数
- **AND** 显示连击提示（如 "2x Combo!"）
- **AND** 增加连击加成分数

#### Scenario: 视觉特效
- **WHEN** 消除元素时
- **THEN** 显示消除动画（元素缩小并消失）
- **AND** 显示粒子特效
- **WHEN** 连击时
- **THEN** 显示连击特效（屏幕震动、光效等）
- **WHEN** 使用特殊道具时
- **THEN** 显示特殊道具特效（爆炸、光线等）

### Requirement: 用户界面
系统应提供清晰直观的用户界面，包括游戏区域、分数显示和控制按钮。

#### Scenario: 主界面布局
- **WHEN** 游戏加载完成
- **THEN** 显示游戏标题
- **AND** 显示当前分数和最高分
- **AND** 显示游戏棋盘
- **AND** 显示重新开始按钮

#### Scenario: 响应式设计
- **WHEN** 在不同设备上访问游戏
- **THEN** 游戏应自适应屏幕尺寸
- **AND** 在移动设备上支持触摸操作
- **AND** 在桌面设备上支持鼠标操作

### Requirement: 状态管理
系统应使用 Pinia 进行状态管理，管理游戏状态、分数和用户数据。

#### Scenario: 游戏状态
- **WHEN** 游戏运行时
- **THEN** 使用 Pinia store 管理棋盘状态
- **AND** 管理分数和连击状态
- **AND** 管理游戏进度状态

#### Scenario: 本地存储
- **WHEN** 游戏结束时
- **THEN** 使用 localStorage 保存最高分
- **AND** 下次游戏时加载最高分
